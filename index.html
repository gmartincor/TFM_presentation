Contexto: Los Bancos de Tiempo

Comprendiendo el ecosistema de intercambio temporal que TimeOverflow gestiona

El Tiempo como Moneda

En los bancos de tiempo, la unidad de intercambio no es el dinero sino las horas. Una hora de enseñanza de inglés equivale exactamente a una hora de cualquier otro servicio, democratizando el valor del tiempo independientemente de la naturaleza del trabajo realizado.

Intercambio de Servicios

Las personas ofrecen sus habilidades y conocimientos a la comunidad, creando un ecosistema de apoyo mutuo basado en la reciprocidad. Desde cuidado de niños hasta reparaciones domésticas, cada servicio tiene el mismo valor temporal.

Fortalecimiento Comunitario

Más allá del intercambio económico alternativo, los bancos de tiempo construyen redes sociales sólidas, fortalecen el tejido comunitario local y promueven la inclusión social mediante la valoración equitativa de todas las contribuciones.

El Desafío Identificado

La limitación fundamental que motivó el desarrollo de este TFM

Comunidades Aisladas

Banco A
Aislamiento
Banco B

Tradicionalmente, los bancos de tiempo funcionaban como islas independientes. Los miembros solo podían intercambiar servicios dentro de su propia organización, desaprovechando el potencial de una red más amplia de habilidades y recursos disponibles en otras comunidades cercanas.

Ecosistema Interconectado

Banco A
Alianza
Banco B

La contribución propuesta permite establecer alianzas formales entre bancos de tiempo, creando una red colaborativa donde los miembros pueden acceder a un catálogo ampliado de servicios y realizar transferencias seguras de tiempo entre organizaciones aliadas.

Desarrollo del Proyecto

Evolución, hitos importantes y desafíos encontrados durante el desarrollo

Fase 1 - Análisis

Estudio del Código Fuente

Análisis exhaustivo de la arquitectura MVC de TimeOverflow, identificación del modelo multi-tenant y comprensión del sistema contable existente.

Fase 2 - Diseño

Conceptualización del Sistema

Diseño del modelo de alianzas interorganizacionales y propuesta inicial a la comunidad mediante issue #789.

Fase 3 - Implementación

Desarrollo de Funcionalidades

Implementación de 4 Pull Requests principales: sistema de alianzas, filtrado de visibilidad, transferencias interbancarias y contacto entre usuarios.

Fase 4 - Expansión

Funcionalidades Adicionales

Desarrollo del mapa geoespacial y sistema de estadísticas en repositorio derivado tras divergencias con los coordinadores principales.

Problemas Encontrados

Infraestructura desactualizada en producción, limitada interacción con coordinadores, y divergencias sobre el alcance de las funcionalidades propuestas.

Soluciones Adoptadas

Creación de repositorio derivado, despliegue independiente en Render, y adaptación de la arquitectura para soportar todas las funcionalidades diseñadas.

Decisiones Técnicas y Estratégicas

Análisis de las decisiones clave que moldearon el proyecto

Decisiones Arquitectónicas

Extensión vs. Reescritura

Decisión de extender el modelo existente en lugar de crear un sistema paralelo, manteniendo la coherencia arquitectónica y facilitando la integración.

Modelo de Transferencias en Tres Pasos

Implementación de transferencias interbancarias mediante tres movimientos contables para garantizar trazabilidad completa y mantener la autonomía de cada banco.

Integración de Leaflet.js

Selección de Leaflet.js en conjunto con OpenStreetMap y su servicio de geocodificación para la visualización geoespacial, priorizando el código abierto y evitando dependencias de servicios propietarios.

Decisiones de Proceso

Metodología Ágil Adaptativa

Uso de Scrum con sprints flexibles para adaptarse a la dinámica del código abierto y los tiempos variables de respuesta de la comunidad.

Fork del Repositorio

Creación de repositorio derivado para implementar funcionalidades adicionales sin comprometer la visión del proyecto principal.

Despliegue en Render

Elección de Render como plataforma de despliegue por su integración con GitHub y capacidad de alojar aplicaciones Rails gratuitamente.

Stack Tecnológico

Ruby on Rails PostgreSQL JavaScript Leaflet.js OpenStreetMap Highcharts Sass Bootstrap JQuery Git GitHub
